### 🐍 Python: Расчет Weighted Multi-Level OBI

Этот скрипт рассчитывает дисбаланс, придавая больший вес ордерам, которые находятся ближе к текущей цене (Spread).

```python
import numpy as np

def calculate_weighted_obi(order_book, levels=10, decay_factor=0.5):
    """
    order_book: dict с ключами 'bids' и 'asks' [[price, size], ...]
    levels: количество уровней стакана для анализа
    decay_factor: коэффициент затухания веса (чем дальше уровень, тем меньше влияние)
    """
    bids = np.array(order_book['bids'][:levels])
    asks = np.array(order_book['asks'][:levels])
    
    # Извлекаем объемы (size)
    bid_volumes = bids[:, 1]
    ask_volumes = asks[:, 1]
    
    # Генерируем веса: [1.0, 0.5, 0.25, 0.125...]
    weights = np.array([decay_factor**i for i in range(levels)])
    
    # Взвешенные объемы
    weighted_bid_vol = np.sum(bid_volumes * weights)
    weighted_ask_vol = np.sum(ask_volumes * weights)
    
    # Формула OBI: (Bid - Ask) / (Bid + Ask)
    obi = (weighted_bid_vol - weighted_ask_vol) / (weighted_bid_vol + weighted_ask_vol)
    
    return round(obi, 4)

# Пример данных из WebSocket Bybit
sample_book = {
    'bids': [[30000.5, 1.2], [30000.0, 5.0], [29999.5, 10.0]],
    'asks': [[30001.0, 0.5], [30001.5, 2.0], [30002.0, 4.0]]
}

current_obi = calculate_weighted_obi(sample_book, levels=3)
print(f"Current Weighted OBI: {current_obi}")
# Если результат > 0: Перевес покупателей
# Если результат < 0: Перевес продавцов

```

import gym
from stable_baselines3 import PPO

class ArbitrageEnv(gym.Env):
    def __init__(self, data_stream):
        super(ArbitrageEnv, self).__init__()
        # State: [OBI, Spread, Gas, Liquidity]
        self.observation_space = gym.spaces.Box(low=-1, high=1, shape=(4,), dtype=np.float32)

        self.data = data_stream

    def step(self, action):
        # 1. Execute simulated action
        # 2. Calculate Reward based on net profit
        # 3. Get next state
        reward = self._calculate_reward(action)
        obs = self._get_next_obs()
        done = False
        return obs, reward, done, {}

## 🛡️ MEV-Guard: Protecting the "Golden Path"

MEV (Maximal Extractable Value) protection allows your bot to bypass the public mempool and send transactions directly to miners/validators.

### 1. How it works: The "Private Tunnel"

Instead of broadcasting your transaction to everyone, you wrap it in a **Bundle** and send it to a **Relay** (like Flashbots).

* **If profitable:** The miner includes it in the block.
* **If price moved (Slippage):** The miner discards it. **You pay zero gas** for failed transactions (on most private relays).

### 2. Integration with your DRL/ML Logic

* **The "Kill-Switch":** If the DRL agent detects that a "Sandwich" attack is mathematically possible on the current path, it signals the MEV-Guard to use a **private bundle**.
* **Priority Fees (The Bribe):** Instead of high gas, you offer a "Tip" to the miner.
* **Logic:** `Profit - Miner_Bribe > 0`.

import duckdb
import pandas as pd

# 1. Establish DuckDB connection (In-memory for max performance)
con = duckdb.connect(database=':memory:')

def calculate_obi_duckdb(order_book_df):
    """
    Uses DuckDB's vectorized engine to calculate Weighted OBI 
    much faster than native Python.
    """
    query = """
    SELECT 
        (SUM(bid_vol * weight) - SUM(ask_vol * weight)) / 
        (SUM(bid_vol * weight) + SUM(ask_vol * weight)) as weighted_obi
    FROM order_book_df
    """
    return con.execute(query).fetchone()[0]

def execute_strategy(market_data):
